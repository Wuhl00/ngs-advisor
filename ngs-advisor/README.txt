NGS Advisor (ngs-advisor)

Overview
- Pragmatic advisor for designing next-generation sequencing (NGS) plans focused on real research goals, budget constraints, and execution clarity.
- Outputs a structured, machine- and human-readable plan with stable anchors, budget tiers (A/B/C), key parameters and conversions, pitfalls and QC lines, decision log, and PubMed query + clickable link.
- Language policy: auto follow user language; default English; bilingual on request. Terminology keeps English abbreviations (WGS, RNA-seq, scRNA, ATAC).

Key Features
- Standardized sections with anchors for parsing:
  - [RECOMMENDATION], [BUDGET_TIERS], [PARAMETERS], [PITFALLS], [QC_LINES], [DECISION_LOG], [PUBMED_QUERY], [PUBMED_URL]
- Parameter conversions:
  - Reads(pairs) ≈ Data_Gb × 1e9 / (read_len × 2)
  - Data_Gb ≈ Reads(pairs) × read_len × 2 / 1e9
  - WGS Data_Gb = Genome_Gb × Target_Depth
- PubMed support:
  - Generates an English query (MeSH/tiab + organism + objective + recent years)
  - Provides a clickable search link via URL-encoded term
- Delivery modes:
  - Structured text by default
  - PDF on request (optional; can be generated by external script)

Human Users: How to Use
1) Read SKILL.md to understand the workflow and anchors.
2) Provide a brief description (2–3 sentences) covering:
   - Research goal (e.g., differential expression, variants, cell typing)
   - Organism/sample type and reference genome availability
   - Budget level (tight / moderate / ample / unsure)
3) Answer at most 2 clarifying questions per round (the advisor asks only the most impactful ones).
4) Receive a structured plan containing:
   - [RECOMMENDATION]: one-sentence route + core reason
   - [BUDGET_TIERS]: A/B/C tiers (only differences and their impact)
   - [PARAMETERS]: read length, data量/reads, library type, repeats/cell counts, plus conversions
   - [PITFALLS]: 1–2 pitfalls with “how to avoid”
   - [QC_LINES]: 2–3 critical QC indicators for acceptance
   - [DECISION_LOG]: assumptions and how changes affect the plan
   - [PUBMED_QUERY] and [PUBMED_URL]: copy/edit the query or click the link
5) If you need a final document to share (PI, sequencing company, grant attachment), request “finalization (PDF)”. You can also paste the structured text into Word/Docs and export PDF.

Copy-ready Skeleton (example to fill)
[RECOMMENDATION]
Route: <assay/tech>; Reason: <goal/sample/budget/delivery>

[BUDGET_TIERS]
A: <minimal config + what it risks or limits>
B: <recommended config + why it’s cost-effective>
C: <higher budget + what improves (certainty/resolution)>

[PARAMETERS]
read_length: <value> | why
data_gb_or_reads: <value> | why
library_type: <value> | why
replicates_or_cells: <value> | why
conversions: Reads(pairs) ≈ Data_Gb × 1e9 / (read_len × 2); etc.

[PITFALLS]
1) <pitfall> → avoid by <action>
2) <pitfall> → avoid by <action>

[QC_LINES]
<indicator>: <explanation of what bad means and what to do>
<indicator>: <explanation>

[DECISION_LOG]
Assumption: <X>; if false → change to <Y> (e.g., depth/repeats/route)

[PUBMED_QUERY]
("RNA-seq"[MeSH] OR "transcriptome sequencing"[tiab]) AND "<organism>"[tiab] AND "<objective>"[tiab] AND ("2022"[pdat] : "2026"[pdat])

[PUBMED_URL]
https://pubmed.ncbi.nlm.nih.gov/?term=<URL_ENCODED_QUERY>

Notes
- All numbers are indicative; confirm with literature and vendor support.
- Do not fabricate PMIDs or records; mark unverified examples explicitly.
- Prefer repeats over depth for bulk RNA-seq when budget is tight; highlight batch/randomization for single-cell and clinical samples.

Claw/Trae Users: How to Use
- Place SKILL.md under .trae/skills/ngs-advisor/.
- The skill is discoverable via its frontmatter:
  - name, description, version, language_policy, triggers, inputs, outputs
- The advisor will produce anchored sections in the specified order so claw can parse and route content:
  - [RECOMMENDATION] → summary card
  - [BUDGET_TIERS] → tiered options UI
  - [PARAMETERS] → parameter table with conversions
  - [PITFALLS], [QC_LINES] → checklists
  - [DECISION_LOG] → assumptions panel
  - [PUBMED_QUERY], [PUBMED_URL] → search module (URL encode term for the link)
- PDF finalization is optional; if needed, invoke a renderer to turn the structured plan into a document. The skill itself does not require code to operate; rendering can be added as a separate step/tool.
